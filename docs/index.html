bar.js Loading Bar

A slim progress bar across the top of the page on every load. Trickles forward during loading, snaps to 100% when ready, then fades out.

Basic

<script src="bar/bar.js"></script>
<!-- default: solid blue, 3px -->

Custom color

<script>
  window.BarConfig = { color: '#f43f5e' }
</script>
<script src="bar/bar.js"></script>

Gradient

<script>
  window.BarConfig = {
    gradient: 'linear-gradient(to right, #f093fb, #f5576c, #4facfe)'
  }
</script>
<script src="bar/bar.js"></script>
Order matters. The window.BarConfig script must come before bar.js.

Options

OptionDefaultWhat it does
color'#3b82f6'Solid bar color
gradientnullCSS gradient string — overrides color
height3Bar thickness in px
duration600Finish + fade-out speed in ms

editor.js Code Editor

Turns any element with class="editor" into a code editor with syntax highlighting. Powered by highlight.js — 190+ languages, auto-detected.

Requires highlight.min.js in the same editor/ folder. The script loads it automatically.

Basic

<div class="editor editor-dark editor-number">
  const hello = () => "world";
</div>
<script src="editor/editor.js"></script>

Force a language

<div class="editor editor-dark" data-lang="python">
  def greet(name):
      return f"Hello, {name}"
</div>

Classes

ClassDefaultWhat it does
editor—Required. Activates the editor
editor-dark—Dark theme
editor-light—Light theme
editor-glowoffSyntax tokens glow with their own color
editor-writeonEditable (default, no need to add)
editor-readoffRead-only, shows a badge
editor-numberoffLine numbers on the left
editor-motionoffAnimated cursor + typed chars fade in

Languages (examples)

javascripttypescriptpython rustgocpp htmlcsssql phpbashjson + 180 more

terminal.js Terminal / Console

A terminal emulator component. Two modes: plain display for read-only output, and terminal-interactive for a real shell where users type commands.

Log mode (read-only)

<div class="terminal" data-title="output">
  $ npm install
  + Installing packages...
  + Done in 3.2s
  ! 2 vulnerabilities found
  ? Run npm audit fix to resolve
</div>
<script src="editor/terminal.js"></script>

Interactive mode

<div class="terminal terminal-interactive" data-prompt="user@host:~$">
  # Welcome! Type help to see commands.
</div>
<script src="editor/terminal.js"></script>

With motion

<div class="terminal terminal-interactive terminal-motion">
  $ echo hello
</div>

Line syntax

PrefixColorMeaning
$blueCommand line (shown with prompt)
>whitePlain output / info
+greenSuccess message
!redError message
?yellowWarning message
#mutedComment / system message

Classes

ClassWhat it does
terminalRequired. Activates the terminal
terminal-interactiveAdds an input row — users can type commands
terminal-motionAnimated blinking caret + output lines fade in

Data attributes

AttributeDefaultWhat it does
data-title"bash"Title shown in the header bar
data-prompt"~$"Prompt prefix shown before commands

Custom commands

<script>
  window.TerminalCommands = {
    greet(args, print) {
      print(`+ Hello, ${args[0] || 'stranger'}!`);
    },
    date(args, print) {
      print(`> ${new Date().toLocaleString()}`);
    }
  }
</script>
<script src="editor/terminal.js"></script>
Built-in commands: clear clears the output. help lists commands. Arrow Up/Down cycles command history.

typing.js Typing Effect

Typewriter animation for any element — headings, paragraphs, code, spans.

Plain

<h1 class="typing">Hello, World.</h1>
<script src="effects/typing.js"></script>

Smooth

<h1 class="typing typing-smooth" data-delay="500">
  Building the future.
</h1>

Classes

ClassWhat it does
typingRequired. Types text with a hard blinking cursor
typing-smoothEach character fades in, cursor pulses smoothly

Data attributes

AttributeDefaultWhat it does
data-speed50ms between characters. Lower = faster
data-delay0ms to wait before starting

underwater.js Atmosphere Effect

A deep underwater atmosphere over the whole page — blue tint, drifting caustic light blobs, backdrop blur, and an animated wave distortion.

Usage

<body class="effect-underwater">
  ...
  <script src="effects/underwater.js"></script>
</body>
Tip: Works best on dark backgrounds. All layers are pointer-events: none — never blocks clicks.

What it does

LayerDescription
color tintDeep blue gradient over the full page
causticsAnimated light blobs drifting on canvas, like light through water
blurSubtle backdrop-filter blur + saturation boost
wave distortionSVG fractal noise making the tint ripple slowly
scroll shiftCaustic blobs shift as you scroll

scroll.js Smooth Scroll

Replaces native scroll with smooth inertia-based scrolling. Content glides to a stop instead of snapping instantly.

Note: Don't use scroll.js on the docs page itself — the docs use native CSS overflow scroll on <main> to avoid conflicts with the fixed sidebar.

Auto-initialize

<body scroll scroll-speed="0.5">
  ...
  <script src="scroll/scroll.js"></script>
</body>

Required CSS

<link rel="stylesheet" href="scroll/scroll.css">

Manual initialize

<script src="scroll/scroll.js"></script>
<script>
  const s = new SimpleScroll({ speed: 0.6, lerp: 0.08 });
</script>

Options

OptionDefaultWhat it does
speed0.5How much each wheel tick scrolls. Higher = faster
lerp0.1Smoothing. Lower = more glide, higher = snappier

Methods

MethodDescription
scrollTo(target, duration)Scroll to a Y number or an element. Duration in ms (default 1000)
destroy()Removes all listeners and restores normal scrolling
// scroll to position
window.simpleScroll.scrollTo(800);

// scroll to element
window.simpleScroll.scrollTo(document.querySelector('#about'));

// destroy
window.simpleScroll.destroy();
